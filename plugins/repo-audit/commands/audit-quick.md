# Quick Audit Scan

Read `${CLAUDE_PLUGIN_ROOT}/phases/shared-preamble.md` for critical rules.

This is a fast, deterministic-only audit. No LLM sub-agents are spawned.
It runs your existing tools and pattern scans, then synthesizes a single report.

---

## Step 1: Explain to User

> **What `/audit-quick` does:**
>
> A fast scan using only deterministic tools — no LLM sub-agents.
> Runs in 30-60 seconds and produces a single summary report.
>
> **Tools it runs** (if available): linters, type checkers, dependency audits,
> code metrics, git history, and pattern pre-scans.
>
> **Output:** `sdlc-audit/reports/QUICK_SCAN.md`
>
> Does NOT modify any of your files. `rm -rf sdlc-audit/` undoes everything.

Ask the user to confirm before proceeding.

## Step 2: Discovery

If `sdlc-audit/data/detection.json` does not exist, read and execute
`${CLAUDE_PLUGIN_ROOT}/phases/discovery.md`.

If it already exists, read it and reuse the detection data.

Report: **[1/3] Discovery complete** — [X] languages, [Y] frameworks, [Z] directories

## Step 3: Pre-Analysis

Read and execute `${CLAUDE_PLUGIN_ROOT}/phases/pre-analysis.md`.

Report: **[2/3] Pre-analysis complete** — summarize tool results

## Step 4: Generate Quick Scan Report

Write `sdlc-audit/reports/QUICK_SCAN.md` with the following sections, drawing
ONLY from deterministic tool output (never fabricate data):

```markdown
# Quick Scan Report

*Generated by repo-audit (quick mode)*

## Overview
- **Repository:** [total files] files, [total dirs] directories
- **Languages:** [list with file counts]
- **Frameworks:** [list]
- **Scan time:** [duration]

## Linter Results
[For each linter that ran, summarize error/warning counts]
[Include the top 10 most common violations with file paths]
[Link to raw output: `sdlc-audit/tool-output/linter-results/`]

## Type Checker Results
[For each type checker that ran, summarize error counts]
[Include the top 10 type errors with file paths]
[Link to raw output: `sdlc-audit/tool-output/typecheck/`]

## Dependency Vulnerabilities
[For each audit tool that ran, list CVEs by severity]
[Link to raw output: `sdlc-audit/tool-output/deps/`]

## Pattern Pre-Scan
[List all pattern matches with counts]
[Highlight critical patterns: hardcoded secrets, eval/exec, SQL injection]

## Code Metrics
[Lines of code by language]
[Largest files]

## Git Activity
[Hotspot files (most-changed)]
[Bus factor per directory]

## Summary
- **Critical issues:** [count from linters + type errors + CVEs + secret patterns]
- **Warnings:** [count]
- **Tools not available:** [list any skipped tools with install commands]

For a deeper analysis including architecture review, cross-module analysis,
and LLM-powered code review, run `/audit` for the full audit.

---
*Generated by repo-audit (quick mode)*
```

**Rules:**
- Only include sections where data was actually collected
- Never fabricate findings — if a tool didn't run, omit that section
- Link to raw data files so users can inspect details

Report: **[3/3] Quick scan complete** — [summary of findings]

Present the summary to the user and suggest `/audit` for deeper analysis if
significant issues were found.
